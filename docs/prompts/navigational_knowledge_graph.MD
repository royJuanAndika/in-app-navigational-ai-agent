# Navigational Knowledge Graph Architecture

This document outlines the structure of the Navigational Knowledge Graph, which is designed to map user intents to UI elements and page navigation within an admin panel. It enables an AI Agent to guide users step-by-step or provide contextual information about the interface.

## 1. Graph Components: Nodes

The graph consists of three primary types of nodes, distinguished by their semantic meaning and color representation in the visualization.

### 1.1 Intent Node (Purple)
Represents the user's goal or intent.
* **Properties:**
    * `id`: Unique identifier for the intent (e.g., `"tambah_karyawan"`).
    * `label`: Human-readable name of the intent (e.g., `"Tambah Karyawan"`).
    * `possible_questions`: Array of strings representing variations of how a user might ask for this intent.
    * `embedding`: Vector representation generated by concatenating all other attributes. Used for semantic search.
* **Cypher Example:**
    ```cypher
    (:Intent {id: "tambah_karyawan", label: "Tambah Karyawan", possible_questions: ["Bagaimana cara menambahkan karyawan?", "Nambah karyawan gimana?"], embedding: [0.88, 0.77, 0.66, ...]})
    ```

### 1.2 Page Node (Blue)
Represents specific pages or routes within the admin panel.
* **Properties:**
    * `id`: The routing path or unique ID of the page (e.g., `"/employee"`).
    * `title`: Title of the page (e.g., `"Daftar Karyawan"`).
    * `desc`: Description of the page's purpose.
    * `embedding`: Vector representation of the page context.
* **Cypher Example:**
    ```cypher
    (:Page {id: "/employee", title: "Daftar Karyawan", desc: "Halaman utama untuk mengelola data karyawan.", embedding: [0.1, 0.2, 0.3, ...]})
    ```

### 1.3 Element Node (Pink)
Represents interactive or structural DOM elements present on the admin panel pages (e.g., buttons, forms, modals).
* **Properties:**
    * `id`: Unique identifier matching the DOM element (e.g., `"btn_add_karyawan"`).
    * `desc`: Description of what the element does or looks like.
    * `embedding`: Vector representation of the element's description.
* **Cypher Example:**
    ```cypher
    (:Element {id: "btn_add_karyawan", desc: "Tombol untuk tambah karyawan.", embedding: [0.11, 0.22, 0.33, ...]})
    ```

## 2. Graph Components: Edges (Relationships)

Edges define how intents, pages, and elements interact with one another.

### 2.1 `[:CONTAINS]`
Defines the hierarchical structure of a page. It links a **Page Node** to the **Element Nodes** that exist within it.
* **Example Pattern:** `(Page)-[:CONTAINS]->(Element)`

### 2.2 `[:TRIGGERS]`
Maps the physical state changes or routing in the UI. It represents causality based on user interaction.
* **Behavior:** Can link an `Element` to another `Element` (e.g., clicking a button opens a modal) or an `Element` to a `Page` (e.g., clicking a link changes the URL route).
* **Example Pattern:** `(Element)-[:TRIGGERS]->(Element|Page)`

### 2.3 `[:HAS_STEP]`
Defines the logical, sequential flow of an action. It links an **Intent Node** to the required **Element Nodes** needed to fulfill that intent.
* **Properties:**
    * `action`: The type of interaction required (e.g., `"click"`, `"input"`).
    * `order`: Integer defining the sequence of the step.
* **Example Pattern:** ```cypher
    (Intent)-[:HAS_STEP {action: "click", order: 2}]->(Element)
    ```

## 3. AI Agent Tool Integration

The AI agent interacts iteratively with the user, leveraging specific tools to query this Knowledge Graph. The agent is explicitly prompted to prioritize `get_workflow_by_intent` to ensure answers remain consistent with established FAQs.

### Tool 1: `get_workflow_by_intent`
* **Input:** Natural language query from the user.
* **Process:** Performs a semantic search on the `Intent` node's embedding. If the confidence score passes a threshold, it fetches the sequential steps.
* **Cypher Query Executed:**
    ```cypher
    MATCH(i:Intent {id: "tambah_karyawan"})-[r:HAS_STEP]->(step)
    RETURN step.id, step.desc, r.order
    ORDER BY r.order ASC
    ```
* **Output Output:** Ordered JSON array of elements mapping out the tutorial/steps.

### Tool 2: `get_page_content`
* **Input:** Page title or ID.
* **Process:** Performs a semantic search to locate the specific `Page` node, then retrieves all elements belonging to that page via the `CONTAINS` relationship.
* **Cypher Query Executed:**
    ```cypher
    MATCH (p:Page {id: "/employee"})-[:CONTAINS]->(e:Element)
    RETURN
    p.id AS PageID,
    p.title AS PageTitle,
    collect({
    element_id: e.id,
    description: e.desc
    }) AS PageContents
    ```
* **Output:** JSON structure detailing the page and its specific contents.

### Tool 3: `Execute Cypher Query in Neo4j`
* **Function:** A fallback/advanced tool that allows the AI agent to execute custom, raw Cypher queries directly against the Neo4j database if the pre-built tools are insufficient for the user's specific request.
